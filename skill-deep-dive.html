How Skills Work in Claude Code

A skill is a SKILL.md file placed in ~/.claude/skills/. When you type a slash command like /eval-faq, Claude Code reads the corresponding SKILL.md and follows its instructions — including what sources to fetch, what output format to use, and what rules to follow. The skill is the prompt that shapes Claude's behavior. Each skill below is a carefully designed prompt with knowledge bases, routing tables, and output templates.

?

/eval-faq

Your AI evaluation knowledge expert

~5 min

What It Does

This skill turns Claude into an AI agent evaluation expert that answers any question about eval methodology. Ask it about grader types, dataset design, non-determinism, scoring, red-teaming — it gives you a direct, opinionated 3–5 sentence answer grounded in authoritative Microsoft sources.

Inputs

Any natural language question about AI evaluation

/eval-faq What makes a good adversarial test case?
/eval-faq How many test cases do I need?
/eval-faq My eval passes 100% — is that good?
Outputs

A concise 3–5 sentence answer with specific numbers and source citations

No fluff, no "it depends." Always resolves to a concrete recommendation.

Layer 3: Answer Style Rules

Strict formatting constraints ensure every answer is useful:

  • 3–5 sentences maximum — no padding, no preamble
  • Opinionated and direct — never "it depends" without immediately resolving
  • Specific numbers — "start with 20-50 cases", "run 3 trials per case"
  • Microsoft sources take priority over external sources
  • Citations at the end — "Per Microsoft's Eval Scenario Library:" / "Per the Triage Playbook:"

Key Concepts You Can Ask About

Microsoft's 4-Stage Framework

Define → Set Baseline & Iterate → Systematic Expansion → Operationalize

Targets: Overall ≥85%, Core business ≥90%, Safety ≥95%, Edge cases ≥70-80%

SHIP / ITERATE / BLOCK

SHIP ≥85%  |  ITERATE 60–84%  |  BLOCK <60%

Safety failures = automatic BLOCK, regardless of overall score

3 Root Cause Types
  1. Eval Setup Issue — the test itself is wrong (~20% of failures!)
  2. Agent Config Issue — the agent needs fixing
  3. Platform Limitation — outside your control
100% Pass Rate = Red Flag

If all your tests pass on the first run, your tests are too easy. Add harder edge cases and adversarial scenarios immediately.

/eval-suite-planner

Design your evaluation plan before writing any test cases

~5 min

What It Does

Give it a plain-English description of your AI agent, and it produces a complete eval suite plan: which scenarios to test, which evaluation methods to use for each, what quality signals matter, and what pass/fail thresholds to target. This is Stage 1 (Define) of Microsoft's 4-stage evaluation framework — the critical step that most teams skip.

Input

A description of your AI agent in plain English:

/eval-suite-planner I am building a
customer support agent for a premium
hand-wash t-shirt product. It answers
care, sizing, storage, and warranty
questions using a product care PDF.
Outputs (3 artifacts)
  1. In-conversation plan — scenario table, quality signals, thresholds, priority order
  2. .docx report — formatted document for stakeholder review
  3. .xlsx spreadsheet — 3 sheets (Scenarios, Quality Signals, Thresholds)

/eval-generator

The output is a table you can review and a CSV file ready to import directly into Copilot Studio.

Inside the SKILL.md — How It Works

Step 1: Input Detection (Two Modes)

The skill automatically detects which mode to use:

Primary Mode

Detects a scenario plan table in the conversation history. Generates one test case per row.

Fallback Mode

No plan found. Accepts a plain-English agent description and generates 6–8 cases from scratch.

Step 2: Test Case Generation Rules

The SKILL.md contains strict rules for generating realistic test cases:

  • Every "Question" must be realistic — specific, not a placeholder. Uses names, dates, product references.
  • Every "Expected Response" must be testable — never vague.
  • For General Quality tests — Expected Response is left blank (LLM judge evaluates without one)
  • For Keyword Match tests — Expected Response contains the required keywords
  • For adversarial cases — Expected Response describes what the agent should NOT do
Copilot Studio Test Set Table Format

The primary output is a markdown table matching Copilot Studio's CSV import format (MS Learn reference):

#QuestionExpected ResponseTesting Method
1How should I wash my t-shirt?(blank for General quality)General quality
2Can I use bleach on this?No. Bleach is prohibited…Compare meaning
3Can I machine wash it?machine washing is not allowed, hand washKeyword match

The CSV file has exactly 3 columns: Question, Expected response, Testing method. Pass scores and additional methods (Tool Use, Custom) are configured in the Copilot Studio UI after import.

5 CSV-Importable Testing Methods + 2 UI-Only

The skill maps scenarios to Copilot Studio's testing methods. 5 methods can be set via CSV import; 2 additional methods are configured in the UI after import.

CSV-importable (set in the Testing method column):

General quality — overall response quality (no expected response needed)
Compare meaning — semantic similarity to expected answer
Similarity — text similarity (phrasing-sensitive)
Keyword match — must-include / must-not-include checks
Exact match — classification labels, structured outputs

UI-only (configure after import in Copilot Studio):

Tool Use — correct tool/topic invocation
Custom — your own evaluation criteria + labels
Step 3: Reviewer Notes (always 3 items)
  1. Most likely to fail — names the hardest test case and explains why
  2. One more scenario to consider — a realistic scenario worth adding manually
  3. Mandatory reminder — "Review every generated scenario. AI-generated test cases need human review."
Output Files
CSV for Copilot Studio Import

Three columns: Question, Expected response, Testing method. Ready for direct import into Copilot Studio Evaluation.

.docx Report for Human Review

Formatted Word document with all test cases, methods, pass scores, and reviewer notes.

Critical: Stay in the Same Session

The generator detects the planner's output in your conversation history. If you close Claude Code between running /eval-suite-planner and /eval-generator, the plan is lost. Run them in the same session, one after the other.

/eval-result-interpreter

Triage eval results and get a SHIP/ITERATE/BLOCK verdict

~5 min

What It Does

Takes Copilot Studio evaluation results — a CSV export, a pasted summary, or even a verbal description — and produces a structured triage report using Microsoft's Triage & Improvement Playbook. It tells you whether to SHIP, ITERATE, or BLOCK, classifies every failure by root cause, and gives you exactly 3 prioritized actions.

Inside the SKILL.md — How It Works

Input Formats (3 options)
Format 1: CSV File

A file path to a Copilot Studio CSV export with columns: question, expectedResponse, actualResponse, testMethodType_1, result_1, passingScore_1, explanation_1

Format 2: Plain Text

Pasted pass/fail counts, list of failures, or verbal description of results

Format 3: + Scenario Plan

Optionally provide the plan from /eval-suite-planner for more accurate category mapping

The 7-Section Output Structure

The SKILL.md enforces a strict 7-section output, each grounded in a specific layer of the Triage Playbook:

0
Pre-triage Infrastructure Check

Were knowledge sources accessible? Any API errors? Auth tokens valid? If infrastructure was unhealthy, failures are marked infrastructure-blocked, not agent-failed.

1
Score Summary

Total cases, passed, failed, pass rate, test methods used, per-method breakdown.

2
Verdict — SHIP / ITERATE / BLOCK

Applied via the Triage Playbook's decision tree with risk-adjusted thresholds.

3
Failure Triage

Each failure gets the 5-question eval verification first (is the TEST wrong?), then classification into 3 root cause types.

4
Explanation Analysis

Parses Copilot Studio's explanation fields and maps patterns to Playbook diagnostic areas.

5
Top 3 Actions

Each follows the pattern: Change X → Re-run Y → Expect Z. Prioritized: safety first, then core business, then lowest-scoring.

6
Pattern Analysis

Checks for 7 cross-signal patterns: concentration, instruction conflicts, over-constraint, etc.

The Decision Tree (embedded in the skill)
ALL safety/compliance cases above ≥95%?
NO → BLOCK Fix safety issues first.
YES → ALL core business cases above ≥80%?
NO → ITERATE Focus on lowest-scoring area.
YES → Capability cases above threshold?
NO → SHIP WITH KNOWN GAPS
YES → SHIP

The 5-Question Eval Verification

Before blaming the agent for any failure, the skill checks if the test itself is wrong:

#Diagnostic QuestionIf YES →
1Is the agent's actual response acceptable?Eval Setup Issue — grader or expected value is wrong
2Is the expected answer still current?Eval Setup Issue — outdated expected answer
3Is the test case realistic?Eval Setup Issue — unrealistic test case
4Could an alternative response also be correct?Eval Setup Issue — grader too rigid
5Is the test method appropriate?Eval Setup Issue — wrong method

At least 20% of failures in a new eval are eval setup issues, not agent issues. Always rule out the test being wrong before blaming the agent.

Cross-Signal Patterns (7 patterns)
PatternLikely Indicates
All failures share "Question not answered"Knowledge source gap or scope issue
Accuracy AND grounding both failingKnowledge source issue (wrong docs retrieved)
Accuracy passing but tone failingRight answer, poor delivery — style instruction needed
Safety passing but accuracy failingAgent over-constrained — review safety restrictions
All failures in one question typeSystemic gap — fix the category, not individual cases
80%+ failures are eval setup issuesPause agent work — fix the evals first
One signal improving, another degradingInstruction conflict (instruction budget problem)

What Makes These Skills Special
  • Grounded in Microsoft's eval framework — not generic advice, but the actual Eval Scenario Library, Triage Playbook, and MS Learn documentation
  • They hand off to each other — the planner's output is automatically detected by the generator
  • They produce tangible artifacts — .docx reports, .xlsx plans, .csv files ready for Copilot Studio import
