Camp AIR — AI Agent Evaluation: Hands-On Exercises

Session block: 0:25–0:50 (25 minutes total) Format: Work individually or in pairs. A facilitator is circulating — raise your hand if you are stuck.

Quick-Reference Cheat Sheet

Keep this visible while you work.

Key skills

Skill Purpose
/eval-faq Ask plain-English questions about eval methodology
/eval-suite-planner Generate a structured eval plan for your agent
/eval-generator Generate a Copilot Studio test set from your plan
/eval-result-interpreter Analyze eval results CSV and get a SHIP / ITERATE / BLOCK verdict

Workflow

Plan (/eval-suite-planner)
  → Generate (/eval-generator)
    → Upload to Copilot Studio  ← skipped in this workshop
      → Interpret (/eval-result-interpreter)

(In this workshop, we skip the upload step and use a pre-built synthetic results CSV instead.)

Copilot Studio test set CSV format

The generated CSV has three columns: Question, Expected response, Testing method. Testing methods: General quality, Compare meaning, Similarity, Exact match, Keyword match. Save as .csv and import into Copilot Studio > Evaluation > New evaluation.

Three debrief questions (answer before 0:50)

  1. What verdict did the interpreter give, and do you agree?
  2. Which failure pattern is most critical to fix first?
  3. What is one test case you would add to your own agent's eval set based on what you learned?

Stage A — Get Oriented

Time box: 5 minutes (0:25–0:30)

You have just watched the live demo. Before you build anything, use /eval-faq to internalize the methodology and /eval-suite-planner to start shaping your eval plan. Stay in this same Claude Code session for all three stages.

Steps

1. If you don't already have Claude Code open, start it now. You will stay in this same session for all three stages.

cd camp-air-eval-harness
claude

2. Run /eval-faq and ask at least two of the questions below. Copy them verbatim or rephrase them in your own words.

/eval-faq What eval scenarios should I use for a RAG agent?

/eval-faq How do I interpret a 75% knowledge grounding score?

/eval-faq What is the difference between business-problem and capability scenarios?

/eval-faq When should I use a model-graded grader instead of a deterministic one?

/eval-faq What makes a good adversarial test case?

/eval-faq How many cases do I need in a dataset to get meaningful signal?

3. As you read each answer, look for: - The names of specific scenario types or grader types mentioned - Any thresholds or rules of thumb (numbers, percentages, counts) - Anything that surprises you or contradicts an assumption you had

4. Pick an agent to evaluate (see the options table in Stage B) and run /eval-suite-planner with a brief description of it. We recommend starting with the T-shirt customer service agent — it's the same one used in the live demo and all the example files.

/eval-suite-planner I am building a customer support agent for a premium hand-wash t-shirt product. It answers questions about product care (washing, drying, ironing), sizing, storage, warranty claims, and escalates issues it cannot resolve. It uses a product care PDF as its knowledge source.

Tip: You do not need to memorize anything. You are building intuition so the next two stages feel less abstract. Keep this session open — the planner output will be used automatically in Stage B.

Checkpoint

Before moving to Stage B, you should be able to answer in one sentence: "For my chosen agent, the most important thing to measure is ___."

Stage B — Generate Your Eval Test Set

Time box: 15 minutes (0:30–0:45)

This is the main building block. You will pick an agent type, generate an eval plan, create a Copilot Studio test set, and review the output — all inside Claude Code using skills.

Step 1 — Choose your agent task

Pick one of the four options below. If you have a real agent you work on, use that instead and adapt the instructions accordingly.

Option Agent description
T-shirt customer service (recommended) Answers customer questions about Premium Hand-Wash T-shirt product care (washing, drying, ironing), sizing, storage, warranty, and escalates issues it cannot resolve. Uses a product care PDF as its knowledge source.
IT helpdesk Troubleshoots VPN issues, password resets, and software installation requests for employees
HR FAQ Answers questions about company policies like PTO, benefits, and onboarding
Sales lead qualification Asks prospects about their needs and routes them to the right sales team

Write down your choice. You will reference it in every prompt you send.

Step 2 — Run /eval-suite-planner

If you already ran /eval-suite-planner in Stage A, skip this step — your plan is already in the session. Otherwise, run it now with a description of your chosen agent.

/eval-suite-planner I am building a customer support agent for a premium hand-wash t-shirt product. It answers questions about product care (washing, drying, ironing), sizing, storage, warranty claims, and escalates issues it cannot resolve.

Review the plan output. It will include recommended scenario categories (happy-path, edge cases, adversarial) and suggested grader criteria.

Step 3 — Run /eval-generator

In the same session, run /eval-generator. It automatically reads the plan you created in the previous step.

/eval-generator

Wait for the output. The generator will produce a Copilot Studio test set as a structured table.

Important: Stay in the same Claude Code session. The generator reads the plan from your conversation context. If you start a new session, you will need to re-run /eval-suite-planner first or re-describe your agent.

Step 4 — Review the generated test set

Read through the generated Copilot Studio test set table. Ask yourself: - Is each test case realistic? Would this actually happen in production? - Is the expected outcome specific and testable, or vague? - Is there a mix of case types: happy-path, edge case, adversarial? - Is there an important scenario that is missing?

Common mistake: Expected outcomes like "the summary should be good" are not testable. Look for specific constraints: a required label, a maximum length, a required topic, or a prohibited phrase.

Step 5 — Save your test set

Select and copy the generated test set table from the Claude Code output into a text file or note. You will not upload it to Copilot Studio during this workshop, but you will have it ready for when you return to your desk.

Tip: The generator also creates a CSV file you can import directly into Copilot Studio > Evaluation > New evaluation. Save that file for later.

Checkpoint

Before moving to Stage C, verify: - You have a complete eval test set ready for Copilot Studio - The test set includes at least three case types: happy-path, edge case, and adversarial - You can describe in one sentence what your test set is checking

Stage C — Interpret Eval Results

Time box: 5 minutes (0:45–0:50)

Since we do not have live Copilot Studio access in this session, you will use a provided synthetic results file to practice interpreting eval output.

Step 1 — Open the synthetic results

Open the file docs/synthetic-eval-results.csv in the cloned repo (camp-air-eval-harness/docs/synthetic-eval-results.csv). It contains simulated pass/fail results for 12 test cases against the T-shirt customer service agent. The columns are:

  • question — the test input sent to the agent
  • expectedResponse — what the agent should have said
  • actualResponse — what the agent actually said
  • testMethodType_1 — the grading method (CompareMeaning, KeywordMatch, GeneralQuality)
  • result_1 — Pass or Fail
  • passingScore_1 — the threshold (0.7 = similarity score, 1 = all keywords found, 4 = general quality rubric)
  • explanation_1 — the grader's reasoning

Note: Copilot Studio exports use CamelCase method names (e.g., CompareMeaning) while the import CSV uses spaces (e.g., Compare meaning). Both refer to the same methods.

Step 2 — Run /eval-result-interpreter

Run the interpreter skill and point it at the synthetic results CSV:

/eval-result-interpreter docs/synthetic-eval-results.csv

The interpreter will analyze the results and produce: - An overall SHIP / ITERATE / BLOCK verdict - A breakdown of pass rates by scenario category - Identified failure patterns and recommendations

Step 3 — Answer the three debrief questions

Write down your answers before the group debrief at 0:50. You will be asked to share.

  1. What verdict did the interpreter give, and do you agree?
  2. Which failure pattern is most critical to fix first?
  3. What is one test case you would add to your own agent's eval set based on what you learned?

Tip: The verdict is a starting point, not a final answer. The interesting question is always: does the failure pattern point to an agent problem, a prompt problem, or a test case problem?

Troubleshooting

Nothing works / I'm completely lost Raise your hand. A facilitator will come to your desk. You can also pair up with a neighbor who has it working — two people, one screen is a perfectly valid way to do this workshop.

Skills not found If /eval-faq, /eval-suite-planner, /eval-generator, or /eval-result-interpreter are not recognized, restart Claude Code (type /exit, then run claude again). Skills load on startup.

Generator does not detect the plan The generator reads the plan from your conversation context. If you started a new Claude Code session, either re-run /eval-suite-planner first or re-describe your agent directly in the /eval-generator prompt.

CSV import issues If the generated CSV has formatting issues when you import it to Copilot Studio, open it in Excel, verify the three columns (Question, Expected response, Testing method), and re-save as CSV.

Auth error when running skills Your Claude Code session may have timed out. Try: (1) type /exit, (2) run claude again. If that doesn't work, ask the facilitator for help.

Stretch Goals

Finished early? Try one or more of these extensions.

Ask /eval-faq deeper questions about your specific results. Take a failure pattern from the interpreter output and ask about it:

/eval-faq My agent is failing on cases where the input contains
multiple topics. What eval strategies help catch this?

Add more scenarios to your plan. Run /eval-suite-planner again with a more specific focus:

/eval-suite-planner Add adversarial scenarios for my t-shirt support
agent where the customer tries to get the agent to make warranty promises
that are not in the product documentation.

Try interpreting the synthetic results with different questions. Ask the interpreter to focus on a specific category or threshold:

/eval-result-interpreter Focus on the adversarial cases only.
What is the pass rate and what are the common failure modes?

Discuss with your neighbor: What would you change in your agent based on the ITERATE or BLOCK verdict? Would you change the prompt, the grounding data, the guardrails, or something else?
